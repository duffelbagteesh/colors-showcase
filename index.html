Defining Colors in CSS

Your Color Adventure

Introduction Named colors RGB Hex codes HSL C3 color formats Display P3 LAB & LCH

CSS (Cascading Style Sheets) has introduced new color formats that allow web designers and developers to create more complex and sophisticated color effects on their web pages.

These new formats include CSS variables, which allow for the creation of dynamic, reusable color themes that can be updated across an entire website with a single change. Additionally, CSS4 has introduced the Lab and LCH color spaces, which provide more precise and predictable color rendering on different devices and screens.

Another new color format is the functional notation, which allows for the creation of custom color functions that can be used to generate complex color schemes and effects. With these new CSS color formats, web designers and developers have more flexibility and control over the appearance of their web pages, making it easier to create visually stunning and engaging user experiences.

We have now gotten to point where we are very familiar with the rgb color format.

The rgb color format is an additive color model, meaning that when you add together different amounts of red, green, and blue light, you can create any color visible to the human eye.

Where 'red', 'green', and 'blue' are values ranging from 0 to 255, representing the intensity of each color component. For example:

We also have been implementing a new notation, with an additional 4th optional value for opacity (alpha) giving us a new and improved RGBA

Where 'alpha' is a value between 0 and 1, representing the opacity of the color. A value of 0 would be completly transparent, while a value of 1 would be completly opaque.

This changed in CSS Colors level 4, which introduces a standardized notation used across newer color formats. rgba() isn't explicitly deprecated, but it's recommended to use the newer format (fortunately, browser support is excellent).

Hex codes

This is probably the most used color format on the web due to them being easy to use and understand, and they can be used in many different contexts, such as HTML, JS, and graphic design software.

Hex codes are a way to represent colors in CSS using a combination of six hexadecimal (base 16) digits. Hexadecimal is a numbering system that uses 16 digits, including the numbers 0-9 and the letters A-F, where each digit represents a value between 0 and 15. It looks like this:

HSL Colors

So far, both of the color formats we have covered are essentially different 'wrappers' on the same fundamental idea of pulling different values for red, green, and blue colors.

Let's look at a completly different one: HSL

This color picker may feel familiar due to being commonly found in graphic design software like Figma or Photoshop

HSL stands for 'Hue, Saturation, and Lightness' it utilizes these 3 attributes to define its colors:

Hue: Is the actual color of the element. It is represented as an angle between 0 and 360 degrees, where 0 and 360 represent red, 120 represents green, and 240 represents blue.

Saturation: refers to the intensity or purity of the color. It is represented as a percentage, where 0% is completly gray (no color), and 100% is the fullest and most vibrant expression of the hue.

Lightness: is the brightness of the color, represented as a percentage, where 0% is black, 50% is the normal color, and 100% is white.

In a way this is really intuitive way to think about color. Instead of controlling the R,G,B light values directly, we've moved to a higher level of abstraction, one more closely aligned with how humans typically think about color.

See the Pen hsl-color-ex by duffelbagteesh (@duffelbagteesh) on CodePen.

One benefit of using HSL colors in CSS is that it allows you to adjust the saturation and lightness of a color without changing the hue. This can make it easier to create consistent color palettes throughout a design, and to make subtle adjustments to the color scheme as needed.

Mordern color formats

So all of the color formats we just refreshed our memories with have been around for a very long time.

Although, recently, we have been working on making some of these new color formats hit the browser market for everyone.

Display P3

So, this is a color space developed by Apple, which is designed to improve the color reproduction of displays, particularly on mobile devices such as iPhones and iPads. It is an extension of the sRGB color space, which is the standard color space used for most web and digital content.

The Display P3 color space has a wider color gamut than sRGB, which means it can represent a larger range of colors. This is particularly noticeable in the green and red regions of the color spectrum, where Display P3 can produce more vibrant and saturated colors than sRGB.

Essentially a color space is a collection of available colors, the palettes we have to pick from. There are millions of possible colors in sRGB, but it doesn't come close to capturing the full range of colors the human eye is capable of seeing.

P3 extends the standard sRGB color space, giving us access to brighter and more vibrant colors.

In CSS, the color space is linked to the color format. If we choose to use the rgb()syntax (or hex codes, or hsl(), we can only ever specify colors in the sRGB color space.

To utilize the P3 color space, we then need to use a different color format: color(display-p3 1 0 0);

The color() function takes a color space, and then a set of RGB values. Instead of ranging from 0 to 255, it uses decimal values from 0 to 1.

Browser support for this function is very low right now, for it is only implemented on Safari

Lab & LCH

LCH (lightness, Chroma, Hue) and Lab (lightness, a, b) are two color spaces used in digital imaging.

Lab is a three-dimensional color space that is designed to be perceptually uniform, meaning that the distance between two colors in the space roughly corresponds to the perceived difference in color by the human eye. Example:

hsl(60deg 100% 50%)
hsl(240deg 100% 50%)

As we can see, both of these colors have the same 'lightness' value of 50%. But they don't look equally light, though. They yellow feels way lighter than the blue.

The HSL color format is modeled after math/physics. It doesn't take human perception into account. And, it turns out, humans don't perceive colors very accurately.

LCH is a color format that aims to be perceptually uniform to humans. Two colors with an equivalent 'lightness' value should feel equally light.

Here is an example of what yellow and blue look like at 55% lightness in LCH:

lch(55% 132 95)
lch(55% 132 280)

Unlike the other color formats we spoke about here, LCH is not bound to sRGB. It isn't even bound to P3. It achieves this by not having an upper limit on chroma.

In HSL, saturation ranges from 0% (no saturation) to 100% (fully saturated). This is possible because we know that we're talking about the sRGB color space, a finite palette of colors.

But LCH isn't linked to a particular color space, and so we don't know where the upper saturation limit is. It's not static: as display technology continues to improve, we can expect monitors to reach wider and wider gamuts. LCH will automatically be able to reference these expanded colors by cranking up the chroma. Talk about future-proofing!
